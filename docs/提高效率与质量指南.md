# 提高执行效率与结果质量指南

## 问题背景

当用户询问"当前股市热点"等需要实时数据的问题时，系统存在两个问题：

1. **质量评估过松** - 缺乏实质性内容的回答被评为"良好"
2. **缺乏实时数据** - 无法获取真实网络数据，只能返回泛泛而谈的内容

## 解决方案

### 1. 严格质量评级标准（已完成）

修改后的评级标准：

| 通过率 | 内容特征 | 评级 |
|--------|---------|------|
| ≥90% | 无实质性缺失 | ⭐⭐⭐⭐⭐ 优秀 |
| ≥85% | 无实质性缺失 | ⭐⭐⭐⭐ 良好 |
| ≥70% | 有具体/示例/数据缺失 | ⭐⭐⭐ 合格（但有实质性内容缺失）|
| <70% | 多方面缺失 | ⭐⭐ 需要改进 |

**警示机制**：当检测到缺乏实质性内容时，显示警示信息。

### 2. 多路径实时数据获取（已完成）

#### 方案架构图

```
用户请求
    │
    ▼
┌─────────────────────────────────────────┐
│  RealtimeFetcher - 实时数据获取器       │
├─────────────────────────────────────────┤
│  路径 1: yfinance → 股票行情            │
│  路径 2: 网页抓取 → 新闻/数据           │
│  路径 3: WebSearch → 网络搜索           │
│  路径 4: LLM → 智能摘要                 │
└─────────────────────────────────────────┘
    │
    ▼
结构化数据 → Agent 使用 → 高质量回答
```

#### 数据源优先级

```
1. 专业 API（yfinance）→ 股票/金融数据
       ↓ 失败
2. 网页抓取（财经网站）→ 新闻/行情
       ↓ 失败
3. 网络搜索（多引擎）→ 通用信息
       ↓ 失败
4. LLM 智能摘要 → 综合整理
```

### 3. 使用方法

#### 快速开始

```python
from simple_agent.tools.realtime_fetcher import RealtimeFetcher

fetcher = RealtimeFetcher()

# 获取股票行情
quote = fetcher.get_stock_quote("600519.SS")
print(quote.summary())

# 获取市场新闻
news = fetcher.get_stock_market_news("A 股", num_results=10)

# 获取市场报告
report = fetcher.get_market_report("A 股")
print(report.summary)

# 网络搜索
results = fetcher.search_web("今日科技新闻", num_results=10)
```

#### 便捷函数

```python
from simple_agent.tools.realtime_fetcher import get_stock, get_market_news, get_market_report

# 快速获取股票
```

### 10. 常见问题

**Q: 为什么有时获取不到实时数据？**

A: 可能原因：
1. 网络连接问题 - 检查网络连通性
2. API 失效 - 已配置多路径降级
3. 目标网站反爬 - 使用 WebSearch 降级

**Q: 如何提高回答质量？**

A: 建议：
1. 开启 DEBUG 模式查看质量评估
2. 使用实时数据获取器获取真实数据
3. 在 Agent 中集成质量检查器

**Q: 质量评级标准能否自定义？**

A: 可以，修改 `cli_agent.py` 中的评级逻辑：
```python
if report.pass_rate >= 0.9 and not has_substance_issues:
    print("⭐⭐⭐⭐⭐ 优秀")
# 调整阈值...
```

---

## 总结

通过以下措施提高执行效率和提高结果质量：

1. **严格质量评级** - 实质性内容缺失的回答不能评为"良好"
2. **多路径数据获取** - 不依赖单一数据源，自动降级
3. **实时数据集成** - 直接从网络获取真实数据
4. **质量评估显示** - DEBUG 模式下可见详细评估

核心理念：**用真实数据说话，不做空洞的回答**
