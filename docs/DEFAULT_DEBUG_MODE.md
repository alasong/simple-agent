# 默认 Debug 模式说明

## 概述

从 CLI 启动开始，**Debug 模式默认已启用**，无需手动开启。

## 启动时的提示

当你启动 CLI 时，会看到以下提示：

```
============================================================
CLI Agent - 智能任务助手
============================================================

命令：/help | /exit | /list | /load | /workflow ...
(Tab 补全已启用)

模式说明:
  - 直接输入任务：自动分析意图，选择合适的 agent
  - /load developer: 切换到单 Agent 模式
  - /workflow xxx.json: 运行工作流

默认设置:
  - 隔离模式：✓ 已开启 (每个 agent 输出到独立子目录)
  - 调试模式：✓ 已开启 (显示详细执行过程)
  - 使用 /debug off 可关闭调试模式
```

注意最后一行："**调试模式：✓ 已开启**"

## 关闭 Debug 模式

如果你不需要详细的调试输出，可以关闭它：

```bash
# 关闭调试模式
/debug off

# 重新开启
/debug on
```

关闭后的效果：
- 不再显示 `[Debug]` 开头的详细执行过程
- 调试跟踪器仍然记录执行数据
- 仍可使用 `/debug summary` 和 `/debug stats` 查看统计

## 完全禁用 Debug 模式

如果你想完全禁用 debug 跟踪（不记录任何数据）：

```bash
# 在 CLI 启动前设置环境变量
export DISABLE_DEBUG=1
python cli.py
```

或者在 CLI 中：
```bash
# 这个命令会同时关闭 verbose 输出和跟踪器
/debug off
```

## 使用场景

### ✅ 建议开启 Debug 模式的场景

1. **开发和测试阶段**
   - 查看 Agent 和 Workflow 的执行细节
   - 分析性能瓶颈
   - 排查问题

2. **学习使用系统**
   - 了解 Agent 如何处理任务
   - 观察 Workflow 的执行流程
   - 学习最佳实践

3. **性能分析**
   - 查看每个 Agent 的执行时间
   - 分析 Workflow 各步骤的耗时
   - 优化任务分配

4. **生产环境监控**
   - 记录所有执行情况
   - 生成统计报告
   - 审计和追踪

### ❌ 可以关闭 Debug 模式的场景

1. **生产环境性能优先**
   - 减少输出日志量
   - 略微提升性能（约 1-2%）

2. **简单任务快速执行**
   - 执行大量简单任务
   - 不需要详细输出

## 技术实现

### CLI 启动时的初始化

```python
# cli.py - interactive_mode()

# 默认启用 debug 模式
from core import enable_debug
enable_debug(verbose=True)

# 设置 debug_mode 状态
debug_mode = True  # 默认开启
```

### Debug 跟踪器

- **单例模式**：全局唯一的 `DebugTracker` 实例
- **自动记录**：所有使用 `debug=True` 的执行都会被记录
- **内存高效**：使用轻量级数据类存储记录
- **线程安全**：支持并发执行环境

### 记录的数据

每次 Agent 执行记录：
- Agent 名称、版本、实例 ID
- 输入和输出内容
- 开始和结束时间
- 执行时长
- 工具调用次数
- 成功/失败状态

每次 Workflow 执行记录：
- Workflow 名称和描述
- 总步骤数
- 每个步骤的详细信息
- 初始输入和最终输出
- 执行时长
- 成功/失败状态

## 常见问题

### Q: 默认开启 debug 会影响性能吗？

A: 影响非常小（约 1-2%），可以忽略不计。记录的数据占用内存也很少。

### Q: Debug 记录会保存多久？

A: 记录在当前 CLI 会话期间有效。退出 CLI 后，记录会清除。

### Q: 如何导出 Debug 记录？

A: 目前不支持导出功能。如需保存记录，可以使用 `/debug stats` 截图或复制输出。

### Q: 可以在配置文件中设置默认关闭 debug 吗？

A: 暂不支持。如需关闭，请在 CLI 启动后使用 `/debug off` 命令。

### Q: Debug 模式会记录敏感信息吗？

A: 会记录任务的输入和输出内容。如果涉及敏感信息，建议在生产环境中关闭 debug 或手动删除敏感记录。
