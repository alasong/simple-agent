# CLI 调试命令集成文档

## 概述

本文档说明如何在 CLI 交互模式中使用调试命令来查看 Agent 和 Workflow 的执行统计信息。

## 新增命令

### `/debug summary`

显示调试执行摘要，包括：
- Agent 执行总次数、成功/失败次数
- Workflow 执行总次数、成功/失败次数
- 总步骤数和步骤成功率
- 平均执行时长

**使用示例：**
```
[CLI Agent] 你：/debug summary
```

**输出示例：**
```
============================================================
调试执行摘要
============================================================

📊 Workflow 执行:
  总次数：3
  成功：3
  失败：0
  总时长：0.001s
  平均时长：0.000s
  总步骤：7
  成功步骤：7
  步骤成功率：100.0%

============================================================
```

---

- Agent 执行的详细统计（按 Agent 分类）
- Workflow 执行的详细统计（按 Workflow 分类）
- 每个 Agent/Workflow 的执行次数、平均耗时、工具调用次数

### `/debug on` | `/debug off`

启用或禁用调试模式。

**使用示例：**
```
[CLI Agent] 你：/debug on
调试模式：已开启
输出目录：./output/cli
调试跟踪器：已启用 (verbose=True)

[CLI Agent] 你：/debug off
调试模式：已关闭
调试跟踪器：已禁用
```

---

### `/debug` (无参数)

切换调试模式状态。

## 使用场景

### 场景 1: 查看 Agent 使用情况

想了解哪些 Agent 被频繁使用，以及它们的性能表现：

```
[CLI Agent] 你：/debug stats
```

查看输出中的"按 Agent 分类"部分，可以看到每个 Agent 的执行次数和平均耗时。

---

### 场景 2: 监控 Workflow 执行

想了解 Workflow 的执行情况和步骤成功率：

```
[CLI Agent] 你：/debug summary
```

快速查看所有 Workflow 的执行统计。

---

### 场景 3: 性能分析

想分析 Agent 和 Workflow 的性能瓶颈：

```
[CLI Agent] 你：/debug stats
```

查看平均耗时，找出执行较慢的 Agent 或 Workflow。

---

### 场景 4: 调试问题

想调试 Agent 或 Workflow 执行过程中的问题：

```
[CLI Agent] 你：/debug on
[CLI Agent] 你：执行某个任务...
```

启用调试模式后，执行任务时会显示详细的调试信息。

## 技术实现

### 核心组件

1. **DebugTracker** (`core/debug.py`)
   - 单例模式跟踪所有执行
   - 记录 Agent 和 Workflow 的执行详情
   - 提供统计信息计算方法

2. **CLI 集成** (`cli.py`)
   - 添加 `/debug summary` 和 `/debug stats` 命令处理
   - 自动补全支持
   - 错误处理和用户提示

3. **Agent 集成** (`core/agent.py`)
   - 在 `run()` 方法中集成调试跟踪
   - 记录执行开始和结束时间
   - 跟踪成功/失败状态

4. **Workflow 集成** (`core/workflow.py`)
   - 在 `run()` 方法中集成调试跟踪
   - 记录每个步骤的执行情况
   - 统计总体成功率和步骤成功率

## 数据记录

### Agent 执行记录

每次 Agent 执行时记录：
- Agent 名称、版本、实例 ID
- 输入和输出内容
- 开始和结束时间
- 执行时长
- 工具调用次数
- 成功/失败状态
- 错误信息（如果有）

### Workflow 执行记录

每次 Workflow 执行时记录：
- Workflow 名称和描述
- 总步骤数
- 每个步骤的详细信息
- 初始输入和最终输出
- 执行时长
- 成功/失败状态
- 步骤成功率

## 常见问题

**Q: 为什么统计信息显示 0 次执行？**

A: 可能是因为：
- 还没有执行任何 Agent 或 Workflow 任务
- 调试跟踪器被禁用（使用 `/debug on` 启用）
- 执行时没有使用 `debug=True` 参数

**Q: 统计信息会保存多久？**

A: 统计信息在当前会话期间有效。重启 CLI 后，统计信息会重置。

**Q: 如何清空统计信息？**

A: 目前不支持手动清空。重启 CLI 即可重置所有统计。

**Q: 统计信息影响性能吗？**

A: 调试跟踪的开销很小，可以忽略不计。但在生产环境中，如果不需要可以关闭调试模式。
