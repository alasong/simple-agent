# 后台任务功能快速开始

## 问题背景

**之前**: 当有任务执行时，UI 输入不可用，必须等待当前任务完成才能提交新任务。

**现在**: 支持后台任务执行，可以同时提交多个任务，UI 不会阻塞。

## 快速上手

### 1. 启动 CLI

```bash
cd /home/song/simple-agent
python cli.py
```

### 2. 使用后台任务

#### 提交后台任务（不阻塞）

## 技术特性

### 并发控制

- **默认并发数**: 3 个任务同时执行
- **队列管理**: 超出并发数的任务自动排队
- **优先级**: 支持任务优先级（暂未开放给用户）

### 状态管理

任务状态流转：
```
pending -> running -> completed
                  |-> failed
                  |-> cancelled
```

### 富文本展示

使用 Rich 库提供美观的表格和面板展示：
- ✅ 成功状态
- ❌ 失败状态
- 🔄 运行中
- ⏳ 等待中
- ⚠️ 已取消

## 注意事项

1. **任务生命周期**: 任务仅在當前会话中有效，重启 CLI 后任务会丢失
2. **超时设置**: `/result` 命令默认最多等待 60 秒
3. **并发限制**: 默认最多 3 个任务同时执行，避免系统过载
4. **错误处理**: 任务失败会显示错误信息，不影响其他任务

## 常见问题

### Q: 后台任务和普通任务有什么区别？

**A**: 
- **普通任务** (`直接输入`): 同步执行，阻塞 UI，必须等待完成
- **后台任务** (`/bg 任务`): 异步执行，不阻塞，可以立即继续输入

### Q: 最多可以提交多少个后台任务？

**A**: 理论上没有限制，但建议同时执行的任务不超过 10 个，避免系统过载。

### Q: 如何知道任务什么时候完成？

**A**: 使用 `/tasks` 命令查看任务状态，当状态变为 `completed`、`failed` 或 `cancelled` 时表示任务结束。

### Q: 取消任务会怎样？

**A**: 正在运行的任务会尽快停止，状态变为 `cancelled`，不会占用系统资源。
